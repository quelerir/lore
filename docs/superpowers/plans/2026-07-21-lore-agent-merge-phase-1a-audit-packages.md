# Lore↔agent-lore Merge — Phase 1a: Promote the audit read-side into canonical packages

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract the already-clean, already-de-Airflow'd audit **read-side** currently vendored inside `lore-core/services/lore-chat/audit/` into two canonical packages — `packages/lore-audit-core` (domain) and `services/lore-audit-api` (HTTP) — set up a uv workspace, and rewire the chat to consume them, with the FileViewer still serving real `lore_core` data.

**Architecture:** The vendored `audit/` is 100% read-only with zero live Airflow imports and all 19 endpoints. Phase 1a is a *restructure of working code*, not new functionality: move domain → `lore_audit`, move HTTP → `lore_audit_api`, make auth and config injected so the API is both mountable (chat) and standalone (uvicorn). The write-side engine/rules/persistence are explicitly **out of scope** (they ride in with the splitter/provider in a later phase).

**Tech Stack:** Python 3.13, uv workspace, FastAPI, Pydantic, psycopg / psycopg_pool, Chainlit (chat only), Docker.

## Global Constraints

- Branch: `lore-agent-merge`.
- **No user-visible behavior change**: FileViewer must still list files and hydrate real chunks from `lore_core` (`10.32.1.65/loreagent_test` over VPN); all 19 audit endpoints preserved; the chat app and its tests keep working.
- **`lore-audit-core` must NOT import** `fastapi`, `pydantic`, `uvicorn`, `chainlit`, or `airflow*`. Its only third-party dep is `psycopg`. It reads **no** config/env — every dependency (DB connection, cursor key, readers) is injected.
- **`lore-audit-api` must NOT import** `chainlit` or `airflow*`. Auth is an **injected** dependency, not a hard import.
- **Canonical env contract (single source of truth).** The audit read DB is the *same physical database* as the chat's TOAST DB. There is exactly ONE env-var name per physical value; no aliases:
  - DB connection: `TOAST_DB_HOST`, `TOAST_DB_PORT`, `TOAST_DB_USER`, `TOAST_DB_PASSWORD`, `TOAST_DB_NAME` (existing names, kept to avoid `.env` churn). Both `lore-chat` config and standalone `lore-audit-api` settings read these exact names. Never introduce `AUDIT_DB_*` aliases.
  - Cursor HMAC key: derived from `CHAINLIT_JWT_SECRET` (one name). In mounted mode the chat derives and injects it; standalone `lore-audit-api` reads the same `CHAINLIT_JWT_SECRET`.
- **Refactor-as-you-move**: split the monoliths during the move (Task 3) per the contracts convention — `enums.py` / `models.py` / `errors.py`; oversized read contracts → `read/{requests,responses,enums,errors}.py`.
- Config style follows the current repo: per-service Pydantic settings, not scattered YAML. No shared config package.

```python
# Mounts the canonical lore-audit-api ASGI app onto the Chainlit FastAPI app.
import hashlib
from lore_audit_api.factory import create_audit_app, build_audit_service
from config import get_settings
from audit_auth import chat_auth_dependency  # cookie (chainlit) + datacraft ticket

def mount_audit(app):
    settings = get_settings()
    cursor_key = hashlib.sha256(settings.chainlit_jwt_secret.encode()).digest()
    service = build_audit_service(dsn=settings.audit_dsn(), cursor_key=cursor_key)
    audit_app = create_audit_app(service=service, auth_dependency=chat_auth_dependency)
    # preserve the position-0 Mount so the SPA catch-all does not shadow it
    app.router.routes.insert(0, __import__("starlette.routing", fromlist=["Mount"]).Mount("/api/v1/audit", app=audit_app))
```
